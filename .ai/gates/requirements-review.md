# 需求分析评审门禁（阶段 1）

**适用：T2 / T3**（T1 用 [change-lite-review](./change-lite-review.md)；T0 无本阶段）。

在**交付需求分析、等待用户确认**时通过。

## 检查清单

- [ ] **问题清晰**：能一句话说清用户痛点 / 目标
- [ ] **范围明确**：有「纳入」与「不纳入」
- [ ] **用户与场景**：谁操作、主路径是什么
- [ ] **约束已知**：鉴权、性能、安全、工期等已列出或标注「待澄清」
- [ ] **待澄清项**：信息缺口已列为问题，而非 Agent 自行假设
- [ ] **产品对齐**：与 FEATURE-PLAN / 文档 AI 平台定位一致
- [ ] **已落盘**：`.ai/planning/<feature-slug>/01-requirements.md` 已写入且可打开

## 产出物（必须）

| 项 | 要求 |
|----|------|
| 路径 | `.ai/planning/<feature-slug>/01-requirements.md` |
| 格式 | 基于 `.ai/templates/feature-brief.md` |
| 确认记录 | 文末 `状态: 待确认` |

禁止仅用对话摘要替代文件。

## 通过后

**停止**，请用户 review 文件并确认：「需求分析是否 OK？确认后进入方案设计。」

用户确认后 → 更新文件中确认记录为「已确认」。

## 未通过 / 用户驳回

修订 `01-requirements.md`；**不要**进入方案设计或写代码。
